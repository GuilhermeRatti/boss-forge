## Schema de recursos lendários (NPC)

Fonte: `module/data/actor/npc.mjs` (schema em ~104–128).

```
system.resources.legact = { max: number, spent: number }
system.resources.legres = { max: number, spent: number }
system.resources.lair   = { value: boolean, initiative: number, inside: boolean }
```

- **`value` NÃO existe no source** de legact/legres — é derivado em `prepareDerivedData` (~443–446): `value = clamp(max - spent, 0, max)`. **Toda escrita deve ir em `spent`.**
- Shim de compatibilidade em `_preUpdate` (~477–481): update em `system.resources.{legact,legres}.value` é convertido para `spent = max(preparado) - value`. Funciona, mas o canônico é `spent`.
- `lair.value` = "tem ações de covil/usos de covil"; `lair.initiative` = iniciativa da lair action clássica (2014); `lair.inside` = toggle "dentro do covil" (2024).

## Covil 2024 é NATIVO (impacta M3)

`prepareDerivedData` (~367–377): `lairAdjustment = Number(lair.value && lair.inside)` soma **+1 em `legact.max` e `legres.max`** (quando max > 0) e ajusta o CR efetivo para XP. Sem gate de versão de regras nesse ajuste (o gate `modernRules` aparece só no label do item de Legendary Resistance, ~448–456). **M3 modo 2024 = togglar `system.resources.lair.inside`; não reimplementar o bônus.**

## Reset de ações lendárias é NATIVO (impacta M1)

- `NPCData.recoverCombatUses(periods, results)` (`npc.mjs` ~544–549): se `periods` contém `"encounter"` **ou** `"turnEnd"` → `results.actor["system.resources.legact.spent"] = 0`.
- Períodos disparados por `Combat5e` (`module/documents/combat.mjs` ~20–131), via overrides do documento (não hooks):
  - `startCombat()` → `encounter`
  - `_onEndTurn(combatant)` → `turnEnd` (do próprio combatente)
  - `_onStartTurn(combatant)` → `turn` + `turnStart`; `_onStartRound()` → `round`; `rollInitiative()` → `initiative`
- `Combatant5e.recoverCombatUses` (`module/documents/combatant.mjs` ~85–142) envolve tudo com hooks **canceláveis**: `dnd5e.preCombatRecovery(combatant, periods)`, `dnd5e.combatRecovery(combatant, periods, results)` e `dnd5e.postCombatRecovery(combatant, periods, message)`.

**Conclusão M1: Boss Forge não reseta `legact` — o sistema já reseta no fim do turno do boss.** Semanticamente equivalente ao "no início do seu turno" da regra 2014, pois nenhuma ação lendária ocorre durante o turno do próprio boss.

## Consumo de ação lendária via activity (impacta M1)

- `CONFIG.DND5E.activityActivationTypes` (`module/config.mjs` ~1053–1077): tipo **`legendary`** tem `consume: { property: "resources.legact" }` e `scalar: true` (custo = `activation.value`). `mythic` idem; `lair` existe sem consumo; `turnStart`/`turnEnd` existem como tipos passivos.
- `ActivityMixin.use()` (`module/documents/activity/mixin.mjs` ~426–441): quando o activation type tem `consume`, monta `config.consume.action = true` → o uso nativo debita `activation.value` de `system.resources.legact.spent`.
- Hooks do fluxo de uso: `dnd5e.preUseActivity` / `dnd5e.postUseActivity`; `dnd5e.preActivityConsumption` / `dnd5e.activityConsumption` / `dnd5e.postActivityConsumption` (canceláveis via `Hooks.call`).

**Conclusão M1: o botão do dialog chama `activity.use()` da activity com `activation.type === "legendary"` — consumo, mensagem de uso e integração com Midi saem de graça.**

## Resistência lendária tem primitivo NATIVO (impacta M2)

- `NPCData.resistSave(message)` (`npc.mjs` ~557–563): valida `message.flags.dnd5e.roll.type === "save"` e ausência de `roll.forceSuccess`; incrementa `legres.spent`; seta `message.flags.dnd5e.roll.forceSuccess = true`.
- Card de chat nativo (`module/documents/chat-message.mjs` `_enrichSaveTooltip` ~637–666): para NPC dono com `legres.value > 0` e save sem sucesso, renderiza botão "resistir" que chama `actor.system.resistSave(message)`. `forceSuccess` também muda a renderização do total (~209–248).
- **Questão aberta para M2**: como isso interage com os cards/saves do Midi-QOL (que rola saves via workflow próprio) — ver `2026-07-03-m2-midi-qol-v13.md`.

## Utilidades notadas de passagem

- `actor.identifiedItems.get("legendary-resistance", { type: "feat" })` — lookup por identifier usado pelo próprio core (`npc.mjs` ~448). Útil para o Designer localizar/gerar features canônicas.
- `NPCData.getLegendaryActionsDescription()` gera o texto padrão de ações lendárias (usado como `legact.label`).
- XP/CR são ajustados pelo covil (`getCRExp(cr + lairAdjustment)`).
